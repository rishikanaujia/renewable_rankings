## How to Switch Modes

### Method 1: .env File (Recommended)

1. Edit `.env` file:
   ```bash
   nano .env
   ```

2. Change the setting:
   ```bash
   # For demo mode
   USE_REAL_AGENTS=false

   # For real agents
   USE_REAL_AGENTS=true
   ```

3. Restart the application:
   ```bash
   python run.py
   ```

### Method 2: Environment Variable (Quick Test)

```bash
# Run once with real agents
USE_REAL_AGENTS=true python run.py

# Run with mock data
USE_REAL_AGENTS=false python run.py
```

## Why Multiple Indicators?

**Redundancy ensures clarity:**

1. **Top Banner** - Immediate visibility when opening the app
2. **About Tab** - Detailed explanation for those who want to understand
3. **Logs** - Technical verification for developers
4. **Color Coding** - Visual distinction at a glance
5. **Icons** - Quick recognition without reading

## User Benefits

âœ… **Transparency** - Always know what data you're viewing
âœ… **Clarity** - Multiple indicators prevent confusion
âœ… **Flexibility** - Easy to switch modes
âœ… **Guidance** - Clear instructions on how to change modes
âœ… **Confidence** - No ambiguity about data source

## Developer Notes

The indicators are implemented in:
- `src/ui/app.py:create_interface()` - Banner and About tab content
- CSS classes: `.data-source-banner`, `.mock-mode`, `.real-mode`
- Environment variable: `USE_REAL_AGENTS` (checked at runtime)

The system is designed to make it **impossible** to be confused about which data source is active.
